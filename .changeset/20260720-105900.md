working-with-bitbucket-api: install `bb` by **copy into `$(brew --prefix)/bin`** instead of symlinking into `~/bin`. Fixes two real-world failures: `~/bin` is not on macOS default PATH (fresh machines got a non-functional install), and symlinks into the version-based plugin cache dangle after every plugin update. The installer now cleans up the legacy `~/bin/bb` symlink, warns when a foreign `bb` shadows the new one on PATH, and the Step 0 gate verifies by version comparison alone (the `readlink` provenance check is gone — a stale copy is caught by comparing `bb --version` against the skill's `BB_VERSION`).
